# Правила форматирования «Красивенько»-экспорта (.xlsx)

Формат вывода для кнопки «Красивенько» (см. `sidebar-toggle.js`, функции
`buildColoredWorkbook` / `addColoredExportButtons`). Ветка `colored-xlsx-export`.

Экспорт строится не из серверного `.xls` (он приходит вообще без форматирования),
а напрямую из уже отрисованной DOM-таблицы отчёта — цвета ячеек берём из
`getComputedStyle(cell).backgroundColor`, дублировать пороги/логику раскраски
не нужно.

## Правила (согласовано 2026-07-16)

1. **Первые 2 столбца исходной таблицы в экспорт не идут** (`ID проверки`,
   `Отчет по проверке` — служебные, второй обычно пустой). Экспорт стартует
   с 3-го столбца исходной таблицы (в файле он становится столбцом 1).
2. **Строка заголовков (1-я строка файла)**: высота 50px, перенос текста
   включён, выравнивание по центру ячейки и по горизонтали, и по вертикали.
3. **2-й столбец файла** (после вырезания п.1 это столбец с «Оценкой»),
   начиная со 2-й строки (то есть данные, не заголовок) — жирный шрифт.
4. **Автоширина столбцов считается ТОЛЬКО по данным**, без учёта текста
   заголовка (у заголовков бывают длинные вопросы анкеты — раздули бы
   колонку до неприличия). Технически: сначала в лист добавляются строки
   данных, автоширина по ним фиксируется, и только ПОСЛЕ этого строка
   заголовка вставляется сверху (`insertRow(1, …)`), не участвуя в замере.
5. **Максимальная ширина столбца — 300px.** Перенос текста включён,
   высота строки для данных — авто (Excel сам растягивает строку под
   перенесённый текст, если явно не задать `row.height`).
6. **Заливка строки заголовка — `#365D8D`, текст — белый.**

## Технические заметки / конвертация единиц

- Excel хранит высоту строки в ПУНКТАХ, не в пикселях: `pt = px * 0.75`
  (стандартная конвертация при 96 DPI). 50px → 37.5pt.
- Excel хранит ширину столбца в «символьных» единицах (число знаков шрифта
  по умолчанию, помещающихся в столбец), не в пикселях. Приближение для
  Calibri 11 (дефолт Excel/ExcelJS): `excel_width = (px - 5) / 7`.
  Обратное: `px ≈ excel_width * 7 + 5`.
- ⚠️ **ExcelJS-специфичный баг** (проверено на 4.4.0): если поставить
  `ws.columns = [...]` ДО того, как в лист добавлены строки — падает
  `TypeError: e.equivalentTo is not a function` внутри `toModel()` при
  `writeBuffer()`. Обходится так: сначала `ws.addRow(...)` для всех строк,
  потом ширина через `ws.getColumn(i).width = …` — ширина, выставленная
  ПОСЛЕ добавления строк, применяется нормально.
- Автовысоты строки в файле НЕТ как записанного значения — это Excel
  сам считает при открытии, если у строки не задан явный `height` и у
  ячеек стоит `alignment.wrapText = true`. Поэтому для авто-высоты просто
  НЕ трогаем `row.height` (в отличие от заголовка, где высота фиксирована).

## Что НЕ входит в эту итерацию (пока не спрашивали)

- Обработка нескольких строк заголовка (только 1 строка TH).
- Отдельные пороги ширины/цвета для конкретных столбцов, кроме 2-го (п.3).
- Заморозка строки заголовка (freeze panes) при скролле — не просили.
